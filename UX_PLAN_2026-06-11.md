# UX 优化计划（2026-06-11 用户旅程走查）

> 依据：app.js / app.py 代码梳理 + 驾驶舱真机走查（桌面 + 移动视口）+ 调仓向导实测。
> 实施约束：守 HANDOFF §3 不变量（统一渲染器 renderWeeklyReport、决策周期单一事实源、执行质量闸、数据诚实）；每批改完 446 测试全绿 + `node --check engine/web/app.js`。

## 0. 结论：不顺畅的三个根因

| # | 根因 | 体感 |
|---|------|------|
| R1 | **长任务无进度**：生成信号(30s–4min)/构建模型组合(60–90s, 可能隐式再刷信号最长4min)/回测(30–120s)/ETF质量刷新(45–120s) 全是同步 HTTP + 转圈文字，无阶段提示、无预计耗时、无取消 | "像卡死了"，每周必经路径上每次都疼 |
| R2 | **校验与反馈时机错位 + 模式不统一**：调仓表单错误到最后一步才爆发；交易前确认清单在第2步、却在第3步点"确认"时才拦（要修的勾选框已不在当前屏）；toast / inline msg / 原生 confirm() ×4 / hint+spin 四种反馈并存 | "被弹回""不知道哪错了" |
| R3 | **信息架构层级深 + 密度不均**：复盘藏在 战略 workspace → 第4个 lens → recordTabs 三层下；战略页"5步流程条"+"4格判断菜单"两套导航并存；ETF 研究一卡一屏（9只=滚9屏） | "找不到""翻不动" |

视觉风格本身（排版/配色/卡片化）是干净的，问题集中在**流程与反馈**，不是皮肤。

## 2. 优化计划（三阶段，可逐批验收）

### 阶段一：体感止血（优先做，预计 2–3 个改动批次）

**P1-1 长任务进度化（对应 R1 / J1-1 / J4-1 / J5-2）**
- 后端：`/api/signals` 改后台线程执行；signals.py 按阶段写进度（预取行情→逐只信号→估值/YTM→旗标→执行质量闸→归档）到 `engine/cache/progress.json`；新增 `GET /api/signals/status` 返回 `{stage, detail, pct, started_at}`。construct / backtest / incumbents 同模式复用一个进度通道（带 task 名）。
- 前端：统一 `runLongTask(label, startFn, statusUrl)`——按钮态 + 卡片内阶段条（"第3步/6：估值取数…已 42s，通常 1–2 分钟"）；construct 隐式刷信号时**显式**显示"正在自动刷新本周信号（最长4分钟）"。
- 验收：生成信号期间能看到阶段名变化≥3次；杀后端进程前端 30s 内报错而非干等 240s。
- 风险：保持"周报与信号逐字节同源"不变量——进度文件不入归档；测试需 mock 状态端点。

**P1-2 调仓向导校验前移 + 确认清单归位（J1-4 / J1-5）**
- 第2步输入即时校验：买入整手(100倍)/份额正数/价格>0/代码非空，行内红字+边框，未通过禁用"下一步"。
- 交易前确认清单移到第3步预览页（确认按钮上方），未勾全则确认按钮禁用+原因提示，不再事后弹回。
- 验收：故意填 150 份买入，第2步立刻标红；不勾清单时"确认完成调仓"按钮灰且说明原因。

**P1-3 反馈统一：自定义确认层替代 4 处 confirm()（R2 / J1-6 / J4-3）**
- 实现一个页面内 `confirmDialog({title, body, danger})`（支持 HTML body）；调仓确认显示成交摘要表；应用大跳变显示 旧权重→新权重 对比表（标方向和幅度）。
- toast 规范化：flash() 增加 warn 级，约定 成功=toast、可恢复错误=inline msg、不可逆动作=confirmDialog。
- 验收：全仓 `grep -c "confirm("` 原生调用归零；应用大跳变弹层能看到每只 旧%→新%。

**P1-4 首屏骨架屏（J2-1）**
- chips 与三卡加 skeleton 占位（CSS 动画即可，不引库）；各 loadXxx 完成后替换。"加载中…"文案具体化（"正在读取最新决策周期…"）。
- 验收：节流网络下首屏无"空白卡"，每张卡有占位形状。

### 阶段二：路径缩短（让该做的事一步直达）

**P2-1 趋势减仓建议 → "带入调仓" 按钮（J1-3，性价比最高的单点改动）**
- trend_alerts 建议块加按钮：打开调仓向导并预填 代码/方向=卖出/金额→折算份额（标"估算，按真实成交修改"）。走既有 `_recheck_cycle_suggestions` 重验，不绕闸门。
- 验收：从建议到第2步表单预填完成 ≤2 次点击，无手抄。

**P2-2 复盘入口提级（J3-1）**
- 方案 A（轻）：首页"调仓记录"卡头部加"历史周报 / 真实业绩 / 月度复盘"快捷链接，点击跳战略 workspace 对应面板。
- 方案 B（重，可选）：records lens 升为顶层第4个 workspace"复盘"。先做 A，观察使用再定 B。
- 验收：从首页到任一复盘面板 ≤2 次点击。

**P2-3 战略页双导航合并（J4-2）**
- 5步流程条为唯一主线；4 格"判断菜单"改为流程下方的"专题入口"分组（或并入对应步骤的展开区），消除"两套并列导航"。
- 验收：首次路径只有一条主线；步骤点击后视区滚动定位到对应面板。
